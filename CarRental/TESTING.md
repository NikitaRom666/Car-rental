# Тестування

## Призначення
У проєкті використовуються unit-тести та негативні сценарії для перевірки правил бронювання і поведінки персистенції.

## Що перевіряється
- Успішне бронювання
- Некоректні дати
- Перетин періодів бронювання
- Недоступне авто
- Null- та empty-ввід
- Поведінка репозиторіїв через замокані залежності

## Запуск тестів
```powershell
dotnet test
```

## Контроль якості
- `dotnet build` має проходити без помилок
- `dotnet test` має проходити повністю
- Правила бронювання перевіряються тестами, а не лише інтерфейсом
