# Ітерація 1: Базова архітектура та use case бронювання

## Що реалізовано
- Доменна модель: Car, Customer, Booking, VehicleCategory, BookingPeriod, AvailabilityStatus
- Контракти для репозиторіїв
- Сервіс бронювання з повною валідацією (`BookingService` + `BookingValidator`)
- DTO для запиту/результату
- Репозиторії з персистенцією через `JsonSerializer` (`FileCarRepository`, `FileBookingRepository`)
- Консольний UI з валідацією вводу, зручним вибором авто, повідомленнями про помилки
- Unit-тести (8+) для всіх сценаріїв
- Документація, діаграми, беклог

## Виявлені проблеми та їх виправлення
- Некоректний парсинг GUID — додано `TryParse` та вибір за номером
- Валідація дат — додано `DateOnly.TryParse`, цикл для повторного введення
- Персистенцію реалізовано через `JsonSerializer`
- Валідацію бізнес-правил винесено у `BookingValidator`

## Покриття тестами
- Успішне бронювання
- Недоступне авто
- Некоректні дати
- Перетин з існуючим бронюванням
- Null/empty параметри
- Покрито всі негативні сценарії
