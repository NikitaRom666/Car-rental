# Фінальний технічний звіт (Final Report)

## Інженерні компроміси та рішення
1. **Збереження даних:** Відмова від повноцінної реляційної СКБД на користь JSON була вимушеним кроком задля забезпечення мобільності лабораторного проєкту без необхідності розгортання серверної інфраструктури на боці перевіряючого.
2. **Рефакторинг:** Перенесення методів зміни стану автомобіля (`Rent()`, `Return()`) безпосередньо в модель `Car` дозволило повністю прибрати логічний smell "Feature Envy" з сервісного шару та досягти високого рівня інкапсуляції.
3. **Паттерни:** Впровадження патерну "Стратегія" дозволило виконати вимогу OCP (Open/Closed Principle) — система відкрита для додавання нових правил розрахунку цін, але закрита для модифікації існуючого коду сервісу.
